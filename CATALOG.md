# Extensions Catalog

A curated index of all extensions in the **Experience Extension Community** organization.

> **How this catalog is maintained:** new extensions are added when promoted via the Promotion-to-Org Checklist (see [ARCHITECTURE.md](./ARCHITECTURE.md) §16). Status changes are made via PR to this file.

## Status Legend

| Badge | Meaning |
|---|---|
| 🟢 **Stable** | Production-ready; passes all promotion criteria; actively maintained |
| 🟡 **Beta** | Functional but may have rough edges; useful as a reference; under active development |
| 🔵 **Experimental** | Proof of concept; not recommended for production |
| ⚫ **Archived** | No longer maintained; kept for historical reference |

## Want your extension listed?

1. Read the [Promotion-to-Org Checklist](./ARCHITECTURE.md#16-promotion-to-org-checklist).
2. Open an [Extension Proposal issue](https://github.com/experience-extension-community/.github/issues/new/choose) to start the conversation.

## Adopting an extension

Click into any repo from the table above. Every extension's `README.md` includes a Quickstart section explaining how to configure it for your institution. Extensions in this org are designed to be plug-and-play — typically you'll only need to edit a branding tokens file and a few config values.

If an extension is missing setup documentation or has institution-specific values hardcoded, open an issue on that repo — we treat that as a bug.
